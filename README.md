# Paper-Code-Template
课题组大/小论文代码归档模板与规范

# 1. 课题组论文代码归档仓库模板与规范

为了保证课题组科研成果的**可复现性**、**可传承性**，所有成员需严格按照本规范整理并上传代码。

## 2. 强制性归档要求

1. **大论文（学位论文）**：所有毕业生的学位论文，其**第三章**和**第四章**的核心实验代码必须上传。
2. **小论文（期刊/会议论文）**：所有以课题组名义发表的小论文，其对应支撑代码必须完整上传。
3. **代码说明**：所有上传的代码，必须在对应的子文件夹中附带 `README.md` 代码说明文档，说明如何运行代码。

## 3. 代码编写与上传规范

- **环境说明**：必须提供完整的环境依赖文件（如 Python的 `requirements.txt` 或 `environment.yml`），或在文档中说明 MATLAB/C++ 等软件的具体版本。
- **数据脱敏**：严禁上传超过 100MB 的大型数据集。若数据涉密，请上传“少量伪造的示例数据（Dummy Data）”以保证代码能够跑通（Demo级别）。
- **注释规范**：
  - 文件头部需注明：作者、日期、代码功能简述。
  - 核心函数/类必须包含 Docstring（输入参数、输出结果的说明）。
  - 复杂的逻辑段必须有行内中文注释。
- **一键运行**：尽量提供一个 `main.py` 或 `run.sh`，确保接手的同学修改数据路径后，可以直接一键跑通。

## 4. 如何使用本模板创建你的仓库？

1. 点击本页面右上角的绿色按钮 **`Use this template`** -> **`Create a new repository`**。
2. 将新仓库命名为 `论文题目-姓名-毕业年份-大论文代码` 或 `论文题目-姓名-年份-期刊名-小论文代码`。
3. 在你的新仓库中，根据实际情况修改/删除不需要的文件夹，填入你的代码并提交。
